# 💼 UI/UX Developer Portfolio

This repository contains a modern, responsive portfolio built with **React** and **Tailwind CSS**, focused on showcasing UI/UX case studies, prototypes, and interactive components.

Key updates in this branch:
- Hero section preserved (visual/layout kept as the primary lead).
- Added a concise UI/UX-focused update plan and case-study structure.

## 🎯 Purpose
Present your design thinking, process, and outcomes clearly: problem → research → design → prototype → results. Emphasize interaction patterns, accessibility, and measurable impact.

## 🔁 Update Plan (phases)
1. Define goals & target audience — clarify role (UI/UX), outcomes to emphasize.
2. Inventory pages & components — capture current screens and reusable UI elements.
3. Select 4–6 case studies — pick diverse, high-impact projects.
4. Draft case-study outlines — each: context, problem, approach, deliverables, outcome.
5. Build reusable case-study component templates — responsive and accessible.
6. Design system refresh — colors, type scale, spacing, tokens.
7. Add micro-interactions & prototypes — Framer Motion or embed Figma prototypes.
8. Accessibility & performance improvements — WCAG checks and Lighthouse scoring.
9. Deploy, measure, and announce — analytics and outreach plan.

## 📚 Case Study Template (suggested)
- Title & role
- Timeline & team
- Problem statement
- Research & insights (user quotes, metrics)
- Ideation & exploration (sketches, wireframes)
- Final designs (screens, states)
- Prototype & interactions (GIF/embedded prototype)
- Outcome & learnings (metrics, links)

## 🔧 Run locally
1. Install dependencies:

```bash
npm install
```

2. Start dev server:

```bash
npm run dev
```

3. Build for production:

```bash
npm run build
```

## 🔧 Tech Stack
- Frontend: React, Vite, Tailwind CSS
- Animations/prototyping: Framer Motion, embedded Figma
- Forms: EmailJS (contact)
- Deployment: Vercel (recommended)
